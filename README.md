# fb-billing-bookmarklet

Bookmarklet для массовой автоматизации Facebook Ads Billing Hub:

1. **Установка адреса аккаунта** (страна, штат, город, ZIP) — через прямой GraphQL-запрос.
2. **Привязка карт** к рекламным аккаунтам — через автозаполнение UI формы Billing Hub. FB-клиент сам шифрует PAN/CVC, мы только подставляем значения в `<input>`.
3. **Make primary** — опционально делает только что привязанную карту основной (через `BillingMakePrimaryStateMutation`).

UI — плавающее окно по центру страницы Facebook, в изолированном iframe (стандартный Ctrl+V работает, FB-перехватчики событий клавиатуры не мешают).

> **Disclaimer.** Проект не аффилирован с Meta / Facebook. Использование автоматизации может нарушать TOS Facebook — ответственность на пользователе. Никаких карточных данных проект не собирает и не отправляет наружу: всё работает локально в браузере, запросы идут только на `facebook.com`. Шифрование PAN/CVC выполняется самим FB-клиентом.

## Возможности

- Адрес устанавливается через `BillingAccountInformationUtilsUpdateAccountMutation` напрямую — без открытия Billing Hub.
- Карты привязываются последовательно в одной рабочей вкладке (не плодит N popup'ов).
- Имя на карте подставляется из `CurrentUserInitialData.NAME` текущего FB-аккаунта.
- Чекбокс "подставлять текущий месяц/год" — формат строки `PAN-CVC`, срок берётся как `текущий месяц + N лет`.
- Чекбокс "сделать новую карту основной" — после привязки проверяет статус карты через `PaymentMethodsListQuery` и, если она не primary, делает её primary через `BillingMakePrimaryStateMutation`.
- Поддержка ZIP-поля если оно появляется на форме карты (некоторые BIN'ы требуют).
- Автоматическое закрытие success-модала ("Card successfully saved").
- Понятный лог по каждой паре `(аккаунт, карта)` — OK / ошибка с причиной.
- Защита от race condition в навигации: после `location.replace()` проверяется что worker реально на нужном `asset_id`, иначе ошибка вместо привязки карты не туда.

## Установка

### Вариант 1: готовая закладка

1. Скачай `set_address.bookmarklet.txt` из релиза или из этого репо.
2. Открой файл в текстовом редакторе, скопируй **всю строку целиком** (начинается с `javascript:`).
3. В Chrome / Edge / любом Chromium-браузере:
   - `Ctrl+Shift+O` (Закладки) → правый клик → **Добавить закладку**
   - **Имя:** любое (например, `FB Billing`)
   - **URL:** вставить скопированную строку
   - Сохранить

### Вариант 2: собрать самому из исходника

```bash
python build_bookmarklet.py set_address.js
```

Скрипт минифицирует JS, URL-кодирует и записывает в `set_address.bookmarklet.txt`. Полезно если хочешь поменять hardcoded адрес или другие константы.

## Использование

### Откуда запускать

- **Set address (этап 1)**: с любой страницы `adsmanager.facebook.com` или `business.facebook.com` где залогинен под нужным актором (Facebook-пользователем).
- **Attach card (этап 2)**: лучше с `business.facebook.com/billing_hub/...`, потому что worker-вкладка открывается на `business.facebook.com`. Запуск с `adsmanager.facebook.com` теоретически работает, но cross-origin блокирует доступ к worker DOM — скрипт выдаст понятную ошибку.

### Этап 1: Set address

1. Открой Facebook (Ads Manager / business.facebook.com).
2. Кликни закладку — появится оверлей по центру страницы.
3. Перейди на вкладку **1. Set address**.
4. Адрес захардкожен в исходнике: `US / DE / Wilmington / 91105`. Если нужны другие значения — правь `ADDRESS` в `set_address.js` и пересобирай через `build_bookmarklet.py`.
5. В textarea — список `act_ID` по строкам (с префиксом `act_` или без):
   ```
   1133545425604191
   act_1694967368416541
   2178639345996169
   ```
6. **Run address** → лог: `[N/M] 1234… → OK` или `→ ✗ <причина>`.

Под капотом — GraphQL мутация `BillingAccountInformationUtilsUpdateAccountMutation` с `doc_id: 33020210320959428`.

### Этап 2: Attach card

1. Перейди на вкладку **2. Attach card**.
2. **Name on card** — предзаполняется именем из FB-профиля. Можно перезаписать вручную.
3. (опционально) Галка **"Подставлять текущий месяц/год"** + поле **Срок (лет)** (по умолчанию 2). Если включено, скрипт сам подставит `MM/YY` = `текущий_месяц / (текущий_год + N лет)`.
4. (опционально) Галка **"Сделать новую карту основной"** — после привязки гарантирует primary-статус через API.
5. Слева — `Account IDs`, справа — карты, соответствующими строками (1:1).

Формат карт:

| Галка "текущий месяц" выкл | Галка вкл |
|----------------------------|-----------|
| `PAN-MM/YY-CVC`            | `PAN-CVC` |

Примеры:
```
4111202020206004-05/28-123
4242424242424242-12/29-456
```
С галкой и сроком 2 года:
```
4111202020206004-123
4242424242424242-456
```

6. **Run cards** → откроется одна worker-вкладка `business.facebook.com/billing_hub/...`. Скрипт по очереди для каждой пары `(account, card)`:
   - переключает worker на нужный `asset_id` (через `location.replace`, с проверкой что URL реально сменился);
   - кликает `Add payment method` → `Debit or credit card` → `Next`;
   - заполняет поля (Name on card, Card number, MM/YY, CVV, ZIP если есть);
   - если включена галка primary и есть поле "Set as default payment method" — кликает его;
   - кликает финальную кнопку (Save / Next / Add — текст зависит от UI-версии);
   - ждёт сообщение "Card successfully saved", закрывает Done;
   - при включённой галке primary — через API проверяет `is_primary` и при необходимости вызывает `BillingMakePrimaryStateMutation`;
   - переходит к следующей паре.

## Технические детали

### Почему адрес — запросом, а карта — через UI

PAN и CVC шифруются клиентским FB-JS перед отправкой (предположительно ECIES P-256; публичный ключ запрашивается отдельной мутацией `PaymentsCometGetServerEncryptionKeyMutation`). В сетевом запросе летят зашифрованные значения `{"sensitive_string_value": "$e2ee"}` + длинный `platform_trust_token`. Воспроизвести шифрование снаружи — это реверс обфусцированного FB-бандла, дорого и хрупко.

Поэтому проще заставить **их же JS** зашифровать через автозаполнение формы. Адрес — обычная GraphQL-мутация без криптографии, поэтому шлётся напрямую.

### Изоляция UI

UI инжектится в `<iframe>` поверх текущей страницы Facebook. Это нужно потому что FB перехватывает keyboard-события на уровне `document` (capture-фазе), включая Ctrl+V — и Shadow DOM от этого не защищает. Iframe-документ полностью изолирован от родителя по событиям.

Окно по центру с полупрозрачным backdrop. В заголовке: `×` — закрыть, `_` — свернуть тело.

### Поиск элементов

Все искалки толерантны к смене UI Facebook:

- **Inputs** — сначала по `name=` (`firstName`, `cardNumber`, `expiration`, `securityCode`, `zip`), fallback на `placeholder` и `aria-label`.
- **Кнопки** — по тексту через `textContent.indexOf`, отсев `aria-disabled="true"` и невидимых элементов (`getBoundingClientRect().width === 0`).
- **Финальная кнопка** — список текстов: `Save`, `Save and continue`, `Next`, `Add card`, `Add`, `Confirm`, `Continue`, `Done`, `Submit`.

### React-валидация ввода

После `setInput` сбрасывается `_valueTracker.setValue('')`. Без этого React-reconciler видит, что value не менялось (с его точки зрения), и при первом rerender переписывает поле обратно в пустое — классическая проблема "ввели и обнулилось" с контролируемыми компонентами.

После заполнения — финальный blur всех полей + перевод фокуса на `body`, чтобы FB-валидация активировала Save (некоторые UI блокируют Save до blur последнего поля).

### Навигация между аккаунтами

`worker.location.replace(url)` (атомарно, без race condition с `href + reload`). К URL добавляется уникальный cache-buster `&_bm=<timestamp>`, чтобы SPA-router FB не решил "URL тот же — ничего не делаю".

После навигации проверяется маркер на старом документе (`worker.__bmNav`): если он исчез — документ перезагружен. Затем явная проверка что `asset_id` в URL равен ожидаемому — если нет, кидаем ошибку вместо привязки к чужому аккаунту.

## Ограничения

- **`doc_id` устаревают.** FB периодически передеплоит UI с новыми persisted-query ID. Когда это случится — увидишь ошибку `Persisted query does not exist` или похожую. Лечится пересниффом нужной мутации (см. ниже).
- **UI-селекторы** для формы карты завязаны на `name=` атрибуты FB. При смене вёрстки потребуется обновить искалки в `attachCard()`.
- **Cross-origin**: для привязки карт запускай bookmarklet с `business.facebook.com` (worker открывается там же).
- **Права актора**: аккаунты должны быть доступны актору (`__user` текущей страницы). Если аккаунт привязан к другому BM/юзеру — вернётся `Application does not have permission`.
- **Antidetect-браузеры** (AdsPower и т.п.) поддерживаются, но `window.open` для worker'а иногда блокируется браузером — разреши popup для `*.facebook.com`.
- **Make primary с CVC.** В некоторых случаях FB может запросить CVC при смене основной карты. В текущей реализации это не покрыто (запрос упадёт с понятной ошибкой). Если столкнёшься — снифай этот flow, открой issue.

## Как обновить `doc_id` (когда что-то перестанет работать)

В этом репо нет sniffer-инструмента, но это несложно сделать через любой HTTP-перехватчик (mitmproxy, или CDP-скрипт на Python+websocket-client).

Что искать:
- `fb_api_req_friendly_name=BillingAccountInformationUtilsUpdateAccountMutation` — для адреса.
- `fb_api_req_friendly_name=BillingHubPaymentSettingsPaymentMethodsListQuery` — для списка карт.
- `fb_api_req_friendly_name=BillingMakePrimaryStateMutation` — для make-primary.

Текущие `doc_id` лежат в `set_address.js` в начале файла как:
```js
var DOC_UPDATE_ACCOUNT       = '33020210320959428';
var DOC_PAYMENT_METHODS_LIST = '26414517794887295';
var DOC_MAKE_PRIMARY         = '24268156329457050';
```

## Безопасность

- Все запросы идут только на `facebook.com`, никаких внешних серверов.
- PAN/CVC шифруются самим FB-клиентом (поле `$e2ee` в payload), bookmarklet их не "видит" — только подставляет в `<input>`.
- Логи в локальный оверлей, никуда не отправляются.

**Тем не менее** — JS-код в закладке имеет полный доступ к вашей facebook-сессии (cookies, dtsg, всё). **Не вставляйте в закладку код, который не понимаете** или который пришёл из недоверенного источника. Этот репо опубликован чтобы код был открыт для аудита; всегда проверяйте `set_address.js` сами или собирайте `.bookmarklet.txt` через `build_bookmarklet.py` из исходника.
